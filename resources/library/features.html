J+ 在技术上相对于其它框架的最大不同点

1. 面向对象

从底层函数到Ajax，从DOM操作到控件操作，整个库是彻彻底底地面向对象的。
面向对象所带来的好处是：

  1. 易维护
        采用面向对象思想设计的结构，可读性高，由于继承的存在，即使改变需求，那么维护也只是在局部模块，所以维护起来是非常方便和较低成本的。
  2. 质量高
        在设计时，可重用现有的，在以前的项目的领域中已被测试过的类使系统满足业务需求并具有较高的质量。
  3. 效率高
        在软件开发时，根据设计的需要对现实世界的事物进行抽象，产生类。使用这样的方法解决问题，接近于日常生活和自然的思考方式，势必提高软件开发的效率和质量。
  4. 易扩展
        由于继承、封装、多态的特性，自然设计出高内聚、低耦合的系统结构，使得系统更灵活、更容易扩展，而且成本较低。
面向对象所带来的缺点：
  1. 学习难度相对大。
    虽然对于本来学习 Java、.Net 等平台语言的用户来说没有压力，但对于广大前端开发人员来说，学习面向对象是一件比较辛苦的事。
  2. 对于单页面，显然面向对象会增加开发的负担。

2. 设计巧妙、速度高、代码量小

正是巧妙的设计，让代码重用度变高，有效地减小学习的成本，减小库文件的体积。 在速度上，力争实现最大效率，不做额外功。

3. 自带的调试系统

也许您曾经遇到过这样一个问题： 引用了一个 Javascript 框架，写好代码之后发现，错误出现在这个框架的源码内部。您不得不去阅读这个框架的源码。 J+ 作为面向对象的库，往往参数的类型都是明确的，所以它可以为每个函数的参数作检查，发现错误时，第一时间报告。此外，一些经常会出错的地方，库里也作了检查。当然，只限于调试版，开发版不会检查。
